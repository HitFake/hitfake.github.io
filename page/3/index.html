MCU嵌入式系统学习摘记

嵌入式系统与MCU的关系

MCU就是一个最典型的嵌入式系统,大部分嵌入式系统以MCU为核心进行设计,因此以MCU为核心的系统是应用最广的嵌入式系统.

学习以MCU为核心的嵌入式系统,需要以下软硬件基础知识与实践训练

①硬件最小系统(电源、晶振、复位、写入调试接口)；
②通用I／O(开关量输入／输出，涉及各种二值量检测与控制)；
③模／数转换(各种传感器信号的采集与处理，如红外、温度、光敏、超声波、方向等)；
④数／模转换(对模拟量设备利用数字进行控制)；
⑤通信(串行通信接口SCI、串行外设接口SPI、集成电路互联总线I2C、CAN、USB、嵌入式以太网、ZigBee技术等)；
⑥显示(LED、LCD等)；
⑦控制(控制各种设备，包括PWM等控制技术)；
⑧数据处理(图形、图像、语音、视频等的处理或识别)；
⑨各种具体应用。

嵌入式系统学习误区

  1. RTOS本身由于种类繁多，实际使用何种RTOS，一般需要由工作单位确定。基础阶段主要学习RTOS的基本原理与RTOS上的软件开发方法，而不是学习如何设计RTOS。
  2. 想成为一名真正的嵌入式系统设计师，在初学阶段，必须打好嵌入式系统的硬件与软件基础。嵌入式系统与硬件紧密相关，是软件与硬件的综合体，没有对硬件的理解就不可能写好嵌入式软件；同样，没有对软件的理解也不可能设计好嵌入式硬件。
  3. 入门选择芯片的原则应为入门时间较短、硬件成本较低,知识要素较多,学习难度较低。

EPPROM实践之24C02

写入过程

  1. 发送器件地址（0XA0）
  2. 发送要写入24C02的内存地址
  3. 发送要写入的数据
  • 发送器件地址的格式
    1 0 1 0 E0 E1 E2 R/W非

高四位1010是24Cxx系列固定器件地址,E由实际连接电平决定,R/W为读写选择

-写功能实现 

void At24c02Write(uchar addr,uchar dat)
{
    I2cStart();
    I2cSendByte(0xa0); //发送写器件地址
    //因为根据上一课sent函数中已包含了等待应答函数,否则需要写在前面
    I2cSendByte(addr); //发送要写入内存地址
    I2cSendByte(dat); //发送数据
    I2cStop();
}

读出过程

  1. 发送写入的器件地址
  2. 发送要读的24C02的内存地址
  3. 发送读出的器件地址
  4. 读取数据

-读功能实现 

uchar At24c02Read(uchar addr)
{
    unsigned char num;
    I2cStart();            //起始信号
    I2cSendByte(0xa0); //发送写器件地址
    I2cSendByte(addr); //发送要读取的地址
    I2cStart();        //初始化I2C总线,表示上面数据已发完
    I2cSendByte(0xa1); //发送读器件地址
    num=I2cReadByte(); //读取数据
    I2cStop();            //总线结束
    return num;            //将读取到的数据反会
}

EEPROM操作注意事项

  1. 写数据必须保证写保护的正确使用
    • 如C中有写保护硬件WP,可以直接接地(即未进行写保护),可直接实现读和写
      某些情况下,为防止误操作,需要多一个io口来控制wp.读的时候拉高,写的时候拉低.
  2. 读写数据的读出写入地址逻辑关系
  3. 通信过程中的时序关系
  4. 读数据最后一个字节必须回应“非应答位”
  5. 通信速率的使用(100k)

E2PROM之静态数码管显示开关机次数

例程

#include<reg51.h>
#include<intrins.h>
#define uchar unsigned char  
#define uint unsigned int

sbit Trig=P1^0;    //触发输入 至少为10us以上
sbit Echo=P1^1;    //输出回响信号  
sbit SDA=P2^1;
sbit SCL=P2^0;
uchar dutab[10]={0x3f,0x06,0x5b,0x4F,0x66,0x6D,0x7D,0x07,0x7F,0x6F};

void I2Cinit();
void I2Cstart();
void I2Cstop();
void ack();
void noack();
void I2Cwr_byte(uchar dat);
uchar I2Cre_byte();
void write_byte(uchar add,uchar dat);//写函数需要地址和数据作为参数
uchar read_byte(uchar add);//需要地址以供返回数据

void delay();
void delay2();

void main()
{
    uchar count;
    count=read_byte(7);
    count=count%10;
    P1=dutab[count];//用静态数码管显示
    count++;
    write_byte(7,count);
    while(1);
}

void delay()
{
    _nop_();
    _nop_();
    _nop_();
    _nop_();
    _nop_();
    _nop_();
    _nop_();
    _nop_();
}

void delay2()
{
    uint a=30000;
    while(a--);
}

void I2Cinit()
{
//主动将SDA和SCL拉高
    SDA=1;
    SCL=1;
}

void I2Cstart()
{
    SDA=1;        
    delay();
    SCL=1;
    delay();//建立时间是SDA保持时间>4.7us
    SDA=0;
    delay();//保持时间是>4us
}

void I2Cstop()//会发现起始和start相反有木有
{
    SDA=0;
    delay();
    SCL=1;
    delay();
    SDA=1;
    delay;
}  

void ack()
{
    uint i;
    SCL=1;
    delay();
    while((SDA==1)&&(i<200))
        i++;
    SCL=0;
    delay();
}

void noack()
{
    SDA=1;
    delay();
    SCL=1;
    delay();
    SCL=0;
    delay();
}

void I2Cwr_byte(uchar dat)
{
    uchar i;
    SCL=0;//时钟线为低电平时才能变化数据,所以数据准备好后才拉高
    for(i=0;i<8;i++)
    {
        if(dat&0x80)//判断高位是否为1
        {
            SDA=1;
        }
        else//高位为0时
        {
            SDA=0;
        }
        dat=dat<<1;
        delay();
        SCL=1;//拉高后芯片开始写入数据
        delay();
        SCL=0;
        delay();
    }
    SDA=1;//释放数据线,芯片才能将应答信号发回
    delay();
}

uchar I2Cre_byte()
{
    uchar i,dat;
    SCL=0;//同写入操作,先拉低才能有变化,拉高后再读取
    delay();
    SDA=1;//释放数据线
    delay();
    for(i=0;i<8;i++)
    {
        SCL=1;//拉高后开始读数据
        dat<<1;
        if(SDA)//同样判读数据线为高电平或低
        {
            dat++;
        }
        SCL=0;
        delay();
    }
    return dat;
}

/*接下来这部分程序的写出得查看时序图*/

void write_byte(uchar add,uchar dat)
{
    I2Cinit();
    I2Cstart();
    I2Cwr_byte(0xa0);//器件地址:24c系列A不变,可编程部分接地所以为0
    ack();
    I2Cwr_byte(10);//选择要写入的地址
    ack();
    I2Cwr_byte(0x55);//试验准备放入发光二极管,这样就是1个亮1个灭...
    I2Cstop();
    delay2();//数据写完后不能立刻读取,要延时一段时间
}

uchar read_byte(uchar add)
{
    /*开始读取数据*/
    uchar store;//存放读到的数据
    I2Cinit();
    I2Cstart();
    I2Cwr_byte(0xa0);
    ack();
    I2Cwr_byte(10);
    ack();
    I2Cstart();
    I2Cwr_byte(0xa1);
    ack();
    store=I2Cre_byte();//将数据放入P0口
    noack();
    I2Cstop();
    return store;
}

形而上学 -- 文字复刻版

一日,子与七七二贤坐而论道子曰

世者万道,皆形而上学也

一贤遂起而问

行而上学

不行...

辍学?

教书育人从古至今都是个值得关注的问题,谈到教育,不得不说到大学生,而你对中国的高等教育真的了解吗?

时至今日,英国在读本科生有239.6万,美国有1993.0417万,而中国在大学生位于世界第一,有2473.1562万.2473万是个怎么样的概念呢?它相遇整个澳大利亚的人口总数再加上中国一个城市的大学生数量.所以看到如此庞大的人群数量,你就真的以为2473万-1个学生在上课?

据调查,中国有98.3%的学生跷过课.假设每堂课27个人中仅有一个人跷课,那么同时在在跷人数也有91.62万.也就是说…

(乱入):"老师,牙买加今天没来上课."

据统计,跷课的学生中有61.1%是觉得上课听不懂,去了也没用;有20.9%认为自学比听课更有效.有12.7是因为参加了社团活动;只有1.7%是因为身体不适…

其他的呢,则是因为一些不可告人的秘密.

但是为什么会有超过一半的学生觉得听不懂呢?到底是因为老师还是学生?至于老师,学生普遍反映老师的教学方式不适应,教学水平不高.注意,我们说的是教学水平,并不是否认教师的能力.教育本来就是一门学问,甚至,教育学还是一个专业.

我们并不否认教学的学识,只是.他们有时并不擅长教学.而且,教授很忙,很辛苦.据了解,大学教授需要每年发表论文1~2篇,同时还有一些各种各样的科学指标.大学教授的工资也并没有想象中高,平均工资为5000元,与应届优秀毕业本科生相当.所以大学教授一大部分收入都来自于在外承包项目.其次大学教授还要评职称,养家糊口,等等等等.以这样的时间比对下来其思考如何教学的时间便必定会减少.so…

(乱入):"教授先生,这门课你挂了."

另一方面,当代大学生也被认为理解力差,不爱思考.在中国,自初中起便一直遵从题海战术,少思考,一样能得高分的思想根深蒂固.同时,学生也很懒很疲惫.他们懂得用各种各样的定理做题,却不好奇其产生的原因.由于中学时填鸭式教育产生的依赖性,当没有人喂时,你要我怎么学习.教育体质框架下,也让学生变得不求善解,但求不挂.

互联网高速发展,让知识变得触手可得,却同时也让学生变得不爱动脑.而思考才是真正进步的阶梯.所以,我们真正需要的是一个善于教育的老师和一个爱思考的学生.

中国3~12岁的小孩里都有其所谓兴趣爱好,绘画,音乐,各类补习班层出不穷.据统计,在中国,每10个小孩中有4个学过美术,2个学过音乐,有6个上过补习班.

(乱入):4+2+6=12,不是只有10个学生吗?

额..有2个可能重复参与了其中2个.

中国的功利化读书也很严重,上名校,读热门专业,做学生工作..似乎从最初目标就已然订好了.从2000初,全民都在学习计算机的计算机热,时至今日,开设计算机相关专业的学校有2045所,而今,全国高校总数也只有2882所.这71.95%的学校真的有实力开设这样一门的专业吗?还是为了有源源不断的学生到来.据了解,甚至有部分纯文科,纯示范的学校也加入其中.而在者,至2008年的金融危机起,报考各院校经济专业雷的学生增加了21.81%.这样,真的好吗?

小时候,你常常会被问到这样一个问题”小朋友,你将来要做什么啊?””我要当一名为伟大的科学家!”

可是,问问现在的自己,还有对真知的纯净渴望吗?当然,我们也不是希望是在将来大家每个都成为伟大的科学家,我们只是希望大学可以是一种兴趣,而不是职业.时至那时,如果你再问我,你将来要做什么啊我会骄傲的回答说:

“我要做一名快乐的卡车司机.”

title: 单片机学习笔记之逐次比较型ADC

tag:单片机

1. 转换方式:直接转换ADC

2. 工作原理

逐次逼近转换过程和用天平称物重非常相似.天平称重物过程是,从最重的砝码开始试放,与被称物体行进比较,若物体重于砝码,则该砝码保留,否则移去.再加上第二个次重砝码,由物体的重量是否大于砝码的重量决定第二个砝码是留下还是移去.照此一直加到最小一个砝码为止.将所有留下的砝码重量相加,就得此物体的重量.仿照这一思路,逐次比较型A/D转换器,就是将输入模拟信号与不同的参考电压作多次比较,使转换所得的数字量在数值上逐次逼近输入模拟量对应值.
